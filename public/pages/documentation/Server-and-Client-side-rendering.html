Server-side rendering

Server-side rendering is where you construct the page on the server before you send it to the client.

Pros and cons

Pros

  • Good for SEO, since it makes it easier for search engines to index your site's pages.
  • Better for people with slow connections, since the server constructs the page, instead of delegating the job to the browser.
  • Faster initial load.

Cons

  • Slower page loading after the first page compared to client-side rendering.
  • A lot of stress on the server, since you are handling all the rendering server-side.

Client-side rendering

Client-side rendering is where you send a barebones html page and some JavaScript that will render the rest of the content.

Pros and cons

Pros

  • Less strain on the server, since the client handles the rendering.

Cons
